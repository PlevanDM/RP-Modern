# 🚀 Техническое задание для Jules от Google - RepairHub Pro

## 📋 Обзор проекта

**RepairHub Pro** - профессиональная платформа ремонта устройств с системой комиссий 10%/5%, расширенной коммуникацией и админ-панелью 2026.

## 🎯 Текущий статус

✅ **Реализовано:**
- Базовая архитектура React + TypeScript + Vite
- Система аутентификации и ролей (client/master/admin)
- Система заказов и предложений
- Расширенная система сообщений
- Система заработков и комиссий
- Правовая система и комплаенс
- Админ-панель 2026
- Мобильная адаптация

❌ **Проблемы для исправления:**
- Избыточные импорты в компонентах (OrderCreationModal, MasterOrderBoard, AdvancedMessaging, FinancialAnalytics)
- Неунифицированный дизайн между компонентами
- Некоторые компоненты не интегрированы в App.tsx
- Отсутствует полное тестирование всех функций

## 🔧 Задачи для Jules

### 1. Исправление импортов (КРИТИЧНО)
**Файлы для исправления:**
- `src/components/OrderCreationModal.tsx` - убрать избыточные импорты из lucide-react
- `src/components/MasterOrderBoard.tsx` - убрать избыточные импорты из lucide-react  
- `src/components/AdvancedMessaging.tsx` - убрать избыточные импорты из lucide-react
- `src/components/FinancialAnalytics.tsx` - убрать избыточные импорты из lucide-react

**Требования:**
- Оставить только необходимые импорты
- Использовать только те иконки, которые реально используются в компоненте
- Проверить все компоненты на наличие неиспользуемых импортов

### 2. Унификация дизайн-системы
**Создать единую дизайн-систему:**
- Стандартные цвета (primary, secondary, success, warning, error)
- Стандартные размеры (xs, sm, md, lg, xl)
- Стандартные компоненты (Button, Card, Input, Modal, Table)
- Стандартные анимации (framer-motion)

**Файлы для создания/обновления:**
- `src/components/ui/DesignSystem.tsx` - основная дизайн-система
- `src/components/ui/Button.tsx` - унифицированная кнопка
- `src/components/ui/Card.tsx` - унифицированная карточка
- `src/components/ui/Modal.tsx` - унифицированное модальное окно

### 3. Интеграция компонентов в App.tsx
**Проверить и исправить:**
- Все новые компоненты должны быть доступны через навигацию
- Модальные окна должны корректно открываться/закрываться
- Состояние приложения должно быть синхронизировано

### 4. Тестирование функций
**Протестировать:**
- Создание заказов клиентами
- Управление заказами мастерами
- Систему сообщений
- Финансовую аналитику
- Админ-панель
- Мобильную версию

## 🛠 Технические требования

### Стек технологий:
- React 18 + TypeScript
- Vite для сборки
- Tailwind CSS для стилей
- Framer Motion для анимаций
- Zustand для состояния
- Lucide React для иконок

### Структура проекта:
```
src/
├── components/
│   ├── ui/                 # Дизайн-система
│   ├── features/           # Функциональные компоненты
│   └── common/             # Общие компоненты
├── services/               # Бизнес-логика
├── store/                  # Управление состоянием
├── types/                  # TypeScript типы
└── utils/                  # Утилиты
```

### Принципы кодирования:
- Компоненты должны быть переиспользуемыми
- Использовать TypeScript строго
- Следовать принципам SOLID
- Писать чистый, читаемый код
- Использовать семантические имена переменных

## 📝 Инструкции по работе

### 1. Клонирование репозитория:
```bash
git clone https://github.com/PlevanDM/RP-Modern.git
cd RP-Modern
npm install
npm run dev
```

### 2. Запуск приложения:
```bash
npm run dev
# Откроется на http://localhost:3002
```

### 3. Тестирование:
```bash
# В консоли браузера доступны функции:
testAllServices()  # Тест всех сервисов
testMessages()     # Тест системы сообщений
testEarnings()     # Тест системы заработков
testCompliance()   # Тест правовой системы
```
